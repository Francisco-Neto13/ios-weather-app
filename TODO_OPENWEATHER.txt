Endpoints oficiais sugeridos
- Geocoding API
  - http://api.openweathermap.org/geo/1.0/direct?q={city name},{state code},{country code}&limit={limit}&appid={API key}
  - Usar para busca de cidade no SearchAdd.
- Current Weather Data
  - https://api.openweathermap.org/data/2.5/weather?lat={lat}&lon={lon}&appid={API key}
  - Usar com units=metric e lang=pt_br ou en, conforme desejado.
- 5 Day / 3 Hour Forecast
  - https://api.openweathermap.org/data/2.5/forecast?lat={lat}&lon={lon}&appid={API key}
  - Serve para forecast, mas é 3 em 3 horas.
- Air Pollution API
  - http://api.openweathermap.org/data/2.5/air_pollution?lat={lat}&lon={lon}&appid={API key}
  - Usar para o card de qualidade do ar.
- Weather Conditions / Icons
  - https://openweathermap.org/weather-conditions
  - Usar weather[0].id e weather[0].icon para mapear a arte local.

Atenção para limitações da OpenWeather
- Current Weather Data não entrega UVI.
- Current Weather Data não entrega dew point.
- Forecast 5 days entrega previsão em passos de 3 horas, não horário real.
- Forecast 5 days cobre 5 dias, não 7 ou 8 dias completos.
- One Call 3.0 entrega current + hourly + daily + alerts, mas a documentação atual indica que ele é subscription only.

Decisões que a outra pessoa vai precisar tomar
- Se o projeto vai usar só endpoints gratuitos:
  - usar current + forecast 5 days + geocoding + air pollution
  - aceitar que o "Hourly Forecast" será, na prática, um "3-hour step forecast"
  - aceitar que o "Weekly Forecast" terá, no máximo, 5 dias ou será agregado manualmente
  - UV Index vai precisar ficar oculto, com fallback, ou vir de outra fonte
  - dew point do card de humidity vai precisar ser calculado manualmente ou removido

Problema importante de ícones/arte
- O set atual de imagens locais não cobre todas as condições da OpenWeather.
- Exemplos de condições sem arte óbvia:
  - clear sky
  - snow
  - mist/fog
  - thunderstorm específico
- Então a outra pessoa vai precisar decidir entre:
  - mapear várias condições para a arte mais próxima
  - adicionar novos assets em public/widgets
  - usar os ícones oficiais da OpenWeather como fallback

Mapeamento sugerido para as artes locais
- Basear no campo weather[0].icon ou weather[0].id.
- Exemplo inicial:
  - 09d / 09n / 10d / 10n -> rain assets
  - 02d / 03d / 04d -> sun-cloud assets
  - 02n / 03n / 04n -> moon-cloud assets
  - extremo/tornado -> tornado asset
- Fallback recomendado:
  - usar o ícone oficial da OpenWeather se não houver arte local equivalente

Observações finais para quem for implementar
- OpenWeather recomenda usar coordenadas para chamadas de clima; nome de cidade deve virar lat/lon via Geocoding API.
- built-in geocoding nos endpoints de clima está marcado como deprecated na documentação atual.
- Quando um campo não existir na resposta, o app precisa ter fallback:
  - rain
  - snow
  - gust
  - visibility
  - air pollution
- Ideal manter formatação centralizada:
  - temperatura
  - hora local da cidade
  - km/h
  - mm de chuva
  - % de chuva
